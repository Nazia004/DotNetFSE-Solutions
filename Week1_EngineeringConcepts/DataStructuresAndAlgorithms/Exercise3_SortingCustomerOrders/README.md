# Exercise 3 – Sorting Customer Orders

## Objective

Sort customer orders by total price using Bubble Sort and Quick Sort.

## Algorithms Used

- Bubble Sort
- Quick Sort

## Time Complexity

### Bubble Sort
- Best Case: O(n)
- Average Case: O(n²)
- Worst Case: O(n²)

### Quick Sort
- Best Case: O(n log n)
- Average Case: O(n log n)
- Worst Case: O(n²)

## Conclusion

Bubble Sort is easy to implement but inefficient for large datasets. Quick Sort is generally preferred because it provides much better average performance.
